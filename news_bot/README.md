# Бот новостей для трейдинга

Присылает в Telegram важные новости по секторам: ИИ, технологии, нефть, газ,
электроэнергетика, финансы/банки, крупные инвестиции. Работает бесплатно
через GitHub Actions — проверяет новости каждые 10 минут, ничего не нужно
держать включённым.

## Шаг 1. Создайте Telegram-бота

1. Откройте в Telegram **@BotFather**.
2. Отправьте `/newbot`, придумайте имя и юзернейм (должен заканчиваться на `bot`).
3. BotFather пришлёт **токен** вида `123456789:AAExampleToken`. Сохраните его.
4. Напишите своему новому боту любое сообщение (например `/start`) — это нужно,
   чтобы бот мог писать вам в ответ.

## Шаг 2. Узнайте свой chat_id

1. Откройте в браузере (подставив свой токен):
   `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates`
2. Найдите в ответе `"chat":{"id":123456789,...}` — это и есть ваш `chat_id`.
   (Если ответ пустой — убедитесь, что вы уже написали боту сообщение, как в шаге 1.4.)

## Шаг 3. Создайте бесплатный репозиторий на GitHub

1. Зарегистрируйтесь на [github.com](https://github.com), если ещё нет аккаунта.
2. Создайте новый **публичный** репозиторий (например `news-bot`).
3. Загрузите в него все файлы из этого архива, сохраняя структуру папок
   (важно, чтобы `.github/workflows/news_bot.yml` остался по этому пути).
   Проще всего — через веб-интерфейс GitHub: "Add file" → "Upload files",
   перетащите все файлы и папки сразу.

## Шаг 4. Добавьте секреты (токен и chat_id)

В репозитории: **Settings → Secrets and variables → Actions → New repository secret**

Добавьте два секрета:
- `TELEGRAM_BOT_TOKEN` — токен из шага 1
- `TELEGRAM_CHAT_ID` — id из шага 2

## Шаг 5. Включите Actions и проверьте

1. Перейдите во вкладку **Actions** репозитория, согласитесь на включение workflow.
2. Откройте workflow **Stock News Bot** → **Run workflow** (кнопка справа),
   чтобы проверить вручную, не дожидаясь расписания.
3. Если всё настроено верно, бот в первый раз напишет вам:
   *"Бот запущен и настроен"* — и дальше уже будет сам присылать важные новости
   каждые ~10 минут.

## Как это работает

- Бот ищет свежие новости через Google News RSS по каждому из секторов
  (бесплатно, без API-ключей).
- Присылает только те новости, заголовок которых содержит "важные" слова
  (слияния, поглощения, IPO, отчётность, регуляторика, крупные суммы,
  рост/падение акций и т.д.) — список редактируется в `bot.py`,
  переменная `IMPORTANT_KEYWORDS`.
- Уже отправленные новости запоминаются (`seen_links.json`), чтобы не дублировать.

## Что можно настроить

- **Секторы и запросы** — переменная `SECTOR_QUERIES` в `bot.py`. Можно добавить
  конкретные тикеры (например `"NVDA stock"`) или убрать ненужные сектора.
- **Частота проверки** — строка `cron: '*/10 * * * *'` в файле workflow.
  `*/5 * * * *` — каждые 5 минут (минимально разумный интервал для бесплатного
  GitHub Actions; точное соблюдение расписания не гарантируется — иногда бывает
  задержка в несколько минут).
- **Ключевые слова важности** — `IMPORTANT_KEYWORDS` в `bot.py`.

## Ограничения, о которых стоит знать

- Это не миллисекундный realtime — проверка раз в 5–10 минут. Для большинства
  трейдинговых решений этого достаточно, но для скальпинга на секундах — нет.
- Фильтр важности — простой по ключевым словам, не нейросетевой анализ.
  Возможны как пропуски, так и лишние сообщения — список слов легко донастроить
  под себя по опыту использования.
- Бесплатный лимит GitHub Actions для публичных репозиториев — не ограничен;
  если сделаете репозиторий приватным, будет лимит ~2000 минут/месяц
  (этого с запасом хватает на проверку каждые 10 минут).
